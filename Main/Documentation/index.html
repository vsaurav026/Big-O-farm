Introduction

Cospo is a creative, modern, professional and responsive Coming soon / Under Construction template, built to introduce your website before launching. The stylish page will indicate that your site is either being worked on or under construction. It is totally universal and will fit any type of website, even thematically specific ones. It was carefully designed to convey perfect elegance through user friendly experience and customization simplicity. Cospo is easy to work with. It comes with working signup form to build your audience and additional informational pages.

Please do not forget to rate Cospo on ThemeForest. This can be done in Downloads area. Just select Cospo in the Download list and rate it. We really appreciate your feedback and will be grateful for your opinion.

Getting Started

This guide is targeted at people, who have at least basic knowledge of html markup language. Furthermore css and javascript knowledge is advised. If you do not feel yourself well with understanding these technologies you should consider finding someone to assist you.

This template can't be installed into any of available CMS (Wordpress, Drupal, etc.).

This documentation will demonstrate the custom elements used in template and help you with their customization.

Installation

Template installation is pretty easy. All required files are in every version so you can delete unused. Here are the required steps:

  1. Unzip the HTML folder
  2. Choose the desired variant
  3. Rename it to index.html
  4. Upload it through FTP

Files Structure

  • /css/ - CSS styles folder
  • /img/ - folder for images
  • /fonts/ - webfont icons folder
  • /video/ - folder with video background options
  • /js/ - JavaScript files folder
  • /php/ - PHP files for your contact/mailchimp form
  • /sass/ - Syntactically Awesome Stylesheets files folder
  • index.html - main HTML file

Custom Scroll

You may have noticed tags with classes .wrapper-scroll, .scroll at the html source. These divs are necessary for the custom scroll fine working .

So, to add custom scroll to your page, you must wrap your content with div .wrapper-scroll and .scroll, like this:

								     
  1. <div class="wrapper-scroll">
  2.     <div class="scroll">
  3.       <!-- Your Content -->
  4.     </div>
  5. </div>

For correct displaying and usage of Bootstrap columns you must also wrap your content with both classes .content and .row div tag.

								     
  1. <div class="wrapper-scroll">
  2.     <div class="scroll">
  3.       <div class="content row">
  4.           <!-- Your Content -->
  5.       </div>
  6.     </div>
  7. </div>

Thereby, code of your page must look like this:

								    
  1. <section class="cospo-page" id="your-page">
  2.       <div class="wrapper-scroll">
  3.         <div class="scroll">
  4.           <div class="content row">
  5.             <!-- Your Content -->
  6.           </div>
  7.         </div>
  8.       </div>
  9. </section>
Centering content

If you want to center your content both vertically and horizontally, you must wrap it with tags .content-center and div, like this: 

								    
  1. <div class="content-center">
  2.         <div>
  3.             <!-- Your Content -->
  4.         </div>
  5. </div>

Cospo Main

Cospo main block is located after header. This block contains all the pages. Each page has a class cospo-page. There are few kinds of pages - home page, simple page, ajax page and blog page. For correct functioning all the pages must have a background block.

Home page has a class home-page and contains logo block, content and footer.

Ajax page contains aside overlay, page title and aside that contains list with your ajax pages that are in folder named /ajax. Ajax requests writing in data-action. Example: data-action="graphic-designer/facts/_1". _1 - your html file without extention.

Blog page has a class blog and contains blog head and blog-post. Each blog post has data-action and data-src with path to the html file in ajax folder. Example: data-action="show-post" data-src="blog/personal-man/photo-post"

Color Skin

You can choose between two color schemes: dark and light. Color of the text and all elements of typography will change accordingly.

To apply color skin, add the class to the tag body, .light for light color skin and the class .dark - for dark one.

Info

If you do not specify a class for the tag body, the default color skin (light) will be applied.

    All data from this form is sent to the email, which can be edited in the file php/contact.php.

    										  
    1. <div class="col-sm-6 col-sm-offset-3 col-md-4 col-md-offset-0">

    6. 			<textarea placeholder="Message" name="comment"></textarea>
    7. 			<div class="form-message"></div>
    8. 			<button type="submit">Send</button>
    9. 		 </form>
    10. 	  </div>
    11. </div>

    The Contact Form offers 6 variants:
    - Txt File
    - Mailchimp
    - GetResponse
    - AWeber
    - Campaign Monitor
    - iContact
    The Txt File is set as the default subscription form.

    To change it, open the file php/notify-me.php and clear appropriate service settings. Also you should insert certain data - APIKEY, USERNAME, LISTID...

    Mailchimp settings
    define('MC_APIKEY', ''); //Your API key from here - http://admin.mailchimp.com/account/api
    define('MC_LISTID', ''); //List unique id from here - http://admin.mailchimp.com/lists/

    GetResponse settings
    define('GR_APIKEY', ''); //Your API key fr om here - https://app.getresponse.com/my_api_key.html
    define('GR_CAMPAIGN', ''); //Campaign name from here - https://app.getresponse.com/campaign_list.html

    AWeber settings
    define('AW_AUTHCODE', ''); //Your Authcode from here - https://auth.aweber.com/1.0/oauth/authorize_app/4ac86d98
    define('AW_LISTNAME', ''); //List name from here - https://www.aweber.com/users/autoresponder/manage

    Campaign Monitor settings
    define('CM_APIKEY', ''); //Your API key from here - https://www.campaignmonitor.com/api/getting-started/#authenticating_with_an_api_key
    define('CM_LISTID', ''); //List id from here - https://www.campaignmonitor.com/api/getting-started/#listid

    iContact settings
    define('IC_APPID', ''); //Your APP ID from here - https://app.icontact.com/icp/core/registerapp
    define('IC_APIUSERNAME', ''); //Your username
    define('IC_APIPASSWORD', ''); //Your password

    Retina

    Retina Ready is a feature for displaying the high resolution images on the modern devices. It is created to show the true view of the HD images.

    In our theme to enable the Retina for images you should have two images:

    • first one has the standard size and a standard name (slide-2.jpg)
    • second one must be twice bigger then the origin and have additional annex @2x (slide-2@2x.jpg

    Both images must be in the same folder!

    If your images are placed in html code via src tag, you need nothing more to do.

    For the images in CSS code, here's the example how you must change CSS:

    1. .part-2 {
    2.   background-image: url('../images/slider/slide-2.jpg');
    3. }
    4. @media (-webkit-min-device-pixel-ratio: 2) {
    5.   .part-2 { background-image: url('../images/slider/slide-2@2x.jpg'); 
    6.   background-size: cover; }
    7. }

    Plugin doesn't affect to the svg images by default. It has no reasons, because svg is a vector format of graphics.
    If you don't want to use the plugin to the inner images, you must add line data-no-retina to the img tag. Example: 

    1. <img src="/img/image.jpg" data-no-retina />
